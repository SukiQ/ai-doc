## 测试验证

### 让 AI 写测试

```
> 为 src/services/user.ts 编写单元测试。用 Vitest，覆盖：正常流程、边界情况、异常处理。
```

```
> 为所有 API 路由写集成测试。用 supertest + Vitest，覆盖：成功响应、参数校验、权限控制、错误处理。
```

### 测试要点

| 场景 | 要覆盖的内容 |
|------|-------------|
| 正常流程 | 核心功能能跑通 |
| 边界情况 | 空值、超长输入、0、负数、特殊字符 |
| 权限控制 | 未认证、无权限、越权访问 |
| 并发冲突 | 重复创建、并发修改 |
| 错误处理 | 数据库异常、外部服务不可用 |

### 运行测试

```
> 运行所有测试，报告结果。失败的逐个分析原因并修复。
```

让 AI 修完一个测一个，不要攒一堆一起修。

## 代码审查

### AI 自查

```bash
/review      # 审查当前变更
```

```
> 审查最近的改动，关注：代码质量、安全隐患、性能问题、遗漏的边界情况。
```

### 代码简化

```bash
/simplify    # 检查代码复用和质量
```

### 安全检查

```
> 检查代码中是否存在：SQL 注入、XSS、敏感信息泄露、未校验的用户输入。
```

重点关注：

| 检查项 | 关注点 |
|--------|--------|
| 输入校验 | 所有外部输入是否经过校验和清洗 |
| 权限控制 | 每个 API 是否检查了认证和授权 |
| 数据安全 | 密码是否加密、敏感数据是否脱敏 |
| 注入风险 | SQL/NoSQL/命令注入、XSS |
| 错误处理 | 是否泄露内部信息（堆栈、SQL） |

## 性能优化

```
> 分析 src/app/api/posts/route.ts 的性能瓶颈，提出优化建议。
```

常见优化方向：

- 数据库查询：N+1 问题、缺失索引、过度查询字段
- API 响应：未分页、未缓存、返回过多数据
- 前端渲染：不必要的重渲染、大包体积、未做懒加载

## 完整流程回顾

| 阶段 | 关键动作 | 容易忽略的 |
|------|----------|-----------|
| 准备 | 环境搭建、CLAUDE.md | 项目结构和规范不写清楚 |
| 开发 | 需求描述、Plan 模式、迭代 | 一次性塞太多需求、不确认就执行 |
| 验证 | 测试、审查、安全检查 | 只测正常流程、跳过安全审查 |
| 交付 | 提交、部署、文档更新 | 忘记更新 CLAUDE.md 和 README |
