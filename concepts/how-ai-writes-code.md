# AI 如何生成代码

本质是 **next token prediction**——基于海量训练数据学到的模式组合，而非真正"理解"算法。

## 理解层次

| 层次 | 能力 | 示例 |
|------|------|------|
| 语法 | 正确的语法结构 | 列表推导式、泛型 |
| 语义 | 理解意图 | 识别阶乘、排序等模式 |
| 上下文 | 结合项目 | 根据已有 Model 和 DB 配置理解查询 |

## 能力边界

**擅长**：语法正确、常见模式、API 调用、单文件逻辑

**局限**：大型项目架构、最新 API、安全漏洞、性能最优解

## 实操要点

- 提供充分上下文（技术栈、已有模式、约束条件）
- 分步引导 > 一次要求完整实现
- 始终审查安全性、边界情况、错误处理
- 用 `/compact` 管理长对话，避免上下文丢失
